SPECYFIKACJA MATERIAŁOWA MM KWIDZYN - DOKUMENTACJA TECHNICZNA

Szczegółowa specyfikacja architektury kodu, zmiennych stanu i renderowania PDF (Wersja: Zintegrowany Kafelkowy UI)

1. Ekosystem i Zależności Plikowe

Aplikacja działa w modelu Client-Side (SPA). Wszystkie operacje na danych, filtrowanie oraz generowanie dokumentów PDF odbywają się bezpośrednio w silniku JavaScript przeglądarki użytkownika bez komunikacji z zewnętrznym serwerem API.

Nazwa pliku Typ architektoniczny Zadania i odpowiedzialności kodu
Start-Uruchom.html Hub / Router aplikacyjny Punkt wejścia systemu. Zawiera skrypt sterujący rozwijaniem interaktywnego kontenera instrukcji dla brygady.
Specyfikacja-Materialowa-Pelna.html Zintegrowany rdzeń (Główny + Dodatek) Zaimplementowana pełna kontrola nad dwiema niezależnymi instancjami formularzy za pomocą przełączania widoków DOM i stanów CSS.
Baza-Edytor.html Moduł administracyjny danych Interfejs do modyfikacji rekordów w tablicy stałej. Odpowiada za czyszczenie znaków ucieczki przed zapisem pliku wyjściowego.

2. Kontrola Stanu i Eliminacja Duplikatów

Aplikacja zarządza zachowaniem interfejsu przy użyciu dwóch globalnych zmiennych rejestrujących stan sesji operacyjnej:

  • STAN_AKTYWNEJ_KARTY (Wartości: "glowna" | "dodatkowa") — Definiuje widoczność odpowiednich paneli formularza oraz przełącza tryb renderowania stylów wydruku.
  • AKTUALNIE_AKTYWNA_KATEGORIA — Przechowuje ciąg tekstowy aktualnie wybranego filtra kategorii sprzętowej z paska narzędzi.

Algorytm wykluczania powtórzeń (Datalist Rebuilder)

Funkcja przebudujDatalistPodKategorie() wykonuje skanowanie sekwencyjne wszystkich pól input nazw przedmiotów w obu widokach formularza, rzutuje ich wartości na małe litery i dodaje do tablicy wykluczeń. Dzięki temu dynamicznie odświeżana lista podpowiedzi nie zawiera duplikatów:

// Pobranie unikalnych wpisów z tabeli głównej (1-55) i dodatkowej (1-25)
if (poleNazwa && poleNazwa.value.trim() !== "") {
    aktualnieUzyteNazwy.push(poleNazwa.value.trim().toLowerCase());
}

3. Architektura Interfejsu Przestrzennego

Interfejs kafelkowy (boxed style) bazuje na pełnym odizolowaniu elementów wejściowych od siatki tabeli strukturalnej. Komórki tabeli posiadają stałe tło, wewnątrz którego renderowane są zaokrąglone kafelki z cieniem pudełkowym.

Dynamiczne zarządzanie klasą .has-data

Zamiast kosztownego ciągłego nasłuchiwania zdarzeń typu MutationObserver, do pól tabeli przypisano wydajne zdarzenia oninput, wywołujące funkcję sprawdzającą zajętość wiersza. W przypadku wykrycia ciągu znaków, klasa .has-data jest wstrzykiwana do elementu nadrzędnego <tr>:

function obslugaZmianyInputu(index) {
    if (nazwa !== "" || jm !== "" || qty !== "" || uwagi !== "") {
        wierszTr.classList.add('has-data');
    } else {
        wierszTr.classList.remove('has-data');
    }
}

Dzięki klasie CSS tr.has-data td input, ramki kafelków stają się wyraźniejsze, a czcionka zmienia wagę na 700, diametralnie poprawiając ergonomię pracy.

4. Przechwytywanie Zdarzenia Drukowania (Wirtualny PDF)

Zintegrowany generator wykorzystuje natywne zdarzenie cyklu życia przeglądarki window.onbeforeprint do przeprowadzenia wirtualnej kompilacji dokumentu przed przekazaniem go do bufora podglądu systemowego drukarki.

Warunkowe przełączanie kontekstu dokumentów

W zależności od stanu zmiennej STAN_AKTYWNEJ_KARTY, skrypt modyfikuje klasę główną znacznika <body>, sterując widocznością bloków przeznaczonych do druku (Załącznik nr 12 oraz Załącznik nr 12a) oraz ukrywając elementy interfejsu ekranowego:

if (STAN_AKTYWNEJ_KARTY === "glowna") {
    document.body.className = "show-print-glowna";
    // Algorytm dynamicznej alokacji stron głównej specyfikacji (Strona 1, 2, 3)
    let totalPages = hasPage3Data ? 3 : (hasPage2Data ? 2 : 1);
    document.getElementById('total-pages-p1').innerText = totalPages;
} else {
    document.body.className = "show-print-dodatkowa";
    // Budowanie struktury druku czystej dokładki (Załącznik 12a)
}
📌 BEZPIECZEŃSTWO I PRYWATNOŚĆ (Decyzja Projektowa):
Kategorycznie zabrania się implementacji automatycznego zapisywania stanów formularza w obiektach LocalStorage/SessionStorage przeglądarki. Z uwagi na obecność danych wrażliwych (numery PESEL, numery rejestracyjne pojazdów), przenoszenie i zapisywanie stanów roboczych dopuszczalne jest wyłącznie za pomocą jawnych, szyfrowanych obiektów w strukturach plików JSON generowanych na żądanie użytkownika.
